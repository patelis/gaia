# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Setup

```bash
uv sync          # install dependencies
cp .env.example .env  # then populate with keys
```

Required `.env` keys:
- `HF_INFERENCE_KEY` — Hugging Face Inference token
- `SUPABASE_URL` / `SUPABASE_SERVICE_KEY` — Supabase vector store
- `TAVILY_API_KEY` — optional, for Tavily web search

## Running

```bash
python app.py    # launches Gradio UI on localhost
```

The Gradio UI requires a Hugging Face login to fetch and submit GAIA benchmark questions.

## Architecture

The agent is a **LangGraph graph** (`agent.py`) with four nodes:

```
START → retriever_node → reranker_node → processor_node ⇄ tools → END
```

1. **`retriever_node`** — Hybrid search: vector search via Supabase RPC + BM25 over `data/metadata.jsonl`, fused with Reciprocal Rank Fusion. Returns up to 20 candidate `task_id`s.
2. **`reranker_node`** — ModernBERT Cross-Encoder re-scores candidates and prepends the top 3 as few-shot context in the message list.
3. **`processor_node`** — Calls `Qwen/Qwen3-32B-Instruct` via HuggingFace Inference API with bound tools. Decides whether to answer directly or invoke a tool.
4. **`tools` (ToolNode)** — Executes the requested tool and loops back to `processor_node`.

**State** (`states.py` → `AgentState`): `messages`, `task_id`, `file_name`, `retrieved_docs`. Messages accumulate via `add_messages` reducer.

**Entry point** (`app.py` → `BasicAgent.__call__`): strips the first 16 characters from the final message content as a post-processing step (`fixed_answer = response['messages'][-1].content[16:]`).

## Key files

| File | Purpose |
|---|---|
| `agent.py` | LangGraph graph definition and all node logic |
| `tools.py` | All `@tool`-decorated functions + `tools_list` |
| `states.py` | `AgentState` TypedDict |
| `utils.py` | `load_config`, `load_prompt`, `init_bm25_index`, `reciprocal_rank_fusion` |
| `app.py` | Gradio UI + `BasicAgent` wrapper |
| `config.yaml` | Model names, retrieval parameters, graph settings |
| `prompts/prompt.yaml` | Main system prompt for the processor node |
| `prompts/vlm_prompt.yaml` | System prompt for the `analyze_image` VLM tool |
| `data/metadata.jsonl` | Local GAIA corpus used for BM25 indexing |

## Configuration

All tunable parameters live in `config.yaml`:
- Toggle retrieval methods: `retrievers.enable_vector_search`, `retrievers.enable_keyword_search`
- Retrieval depth: `retrievers.vector_store.k`, `retrievers.bm25.k`
- Model swap: `models.llm.model_name`, `models.embeddings.model_name`, `models.reranker.model_name`
- Graph limits: `graph.recursion_limit`
- Model cache: `models.cache_folder` (default `./models/hf_cache`)

## Adding tools

1. Define a new `@tool`-decorated function in `tools.py`
2. Add it to `tools_list` at the bottom of `tools.py`

The `read_file` dispatcher tool routes by file extension — add new extensions there too if needed.

## Supabase vector store

The vector store uses a Supabase RPC function named `match_documents` on a table called `gaia_documents`. Embeddings are generated by `Alibaba-NLP/gte-modernbert-base`. The `create_vector_database.ipynb` notebook populates this table.
